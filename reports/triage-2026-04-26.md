## Bugs & regressions

### B1 — Recipe creation fails: `id` missing in Prisma payload — **P0**
- **Source**: telemetry error 2026-04-20T14:04 + AI insight (Tue Apr 21).
- **Repro**: Open Recipe editor, create a new recipe (any name), click Save. The
  POST `/api/recipes` request fails because `routes/recipes.ts:205` reads
  `body.id` and the frontend payload at `public/js/recipe-editor.ts:760-784`
  never sends one. The Recipe Prisma model declares `id String @id` with no
  `@default`, so Prisma rejects the create.
- **Affected users**: anyone creating a new recipe. Confirmed loss: "pupkin
  veggie stew" by info@testtafel.nl.
- **Severity**: **P0**. Silent data loss on a primary write path.
- **Root cause**: missing UUID generation. Other entities generate the id on the
  backend (e.g. `routes/batches.ts:33` uses `crypto.randomUUID()`), but the
  recipe POST does not.
- **Fix**: in `routes/recipes.ts` POST handler, default `id` to
  `crypto.randomUUID()` when `body.id` is missing.

### B2 — Cannot move a batch to Frozen from the Dashboard inventory modal — **P1**
- **Source**: feedback 2026-04-20T14:13 (Daan): "cannot move dahl to frozen
  stock", screen=planner.
- **Repro**: Dashboard → "🍽️ Cooked Food Inventory" → modal lists each cooked
  batch, with a storage badge next to the name. The badge is rendered via
  `storageBadge()` (`public/js/planner.ts:1008`) which produces a static
  `<span class="badge ...">Gastro</span>` with no click handler. In the planner
  / dish list views the badge uses `storageBadgeClass()` plus
  `onclick="cycleStorage(...)"`, which is the discoverable way to flip
  Gastro → Frozen → Vac-packed. From the dashboard inventory modal there is no
  way to reach that flow short of leaving the modal and digging into the
  planner — so users (Daan) report it as broken.
- **Severity**: **P1**. Daily food-safety workflow (move stale stock to freezer
  before HACCP cutoff). Also reinforced by AI insight "Stale batch 'Coconut red
  curry soup' has 30 portions in stock with no future services" — staff aren't
  freezing stale food because the click target is missing.
- **Fix**: make the storage badge in the dashboard inventory modal clickable
  with the same `cycleStorage` handler used elsewhere, and re-render the modal
  on change so the new label appears.

### B3 — "+ Add note" button on expanded batch tile does nothing visible — **P1**
- **Source**: feedback 2026-04-20T14:05 (TestTafel): "add note function not
  working", screen=planner.
- **Repro**: Open a batch tile (planner). Click "+ Add note". Nothing visibly
  happens. Click again — still nothing.
- **Root cause**: `public/js/dishes.ts:241-244` `showNoteInput()` sets
  `b.note = ''` and re-renders. The render condition at line 369 is
  `hasNote = d.note !== undefined && d.note !== ''`. Empty string fails the
  check, so the renderer keeps showing the same "+ Add note" button instead of
  switching to the editable input. The note input never appears.
- **Severity**: **P1**. Reported as broken by an active user. Trivial fix.
- **Fix**: relax `hasNote` to `d.note !== undefined`, so the empty-string
  "ready to type" state renders the text input.

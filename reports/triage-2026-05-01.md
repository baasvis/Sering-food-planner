## Bugs & regressions

### B5 — `e.target?.closest is not a function` — recurring TypeError — **P1**
- **Source**: telemetry 2026-04-30T15:26 (2 occurrences), AI insight #180 (warning).
- **Affected user**: info@desering.org
- **Repro**: Click on a batch tile or Hanos button — if the click lands on a
  Text node or SVG element inside the tile, `e.target` is not an `Element` and
  `.closest()` doesn't exist.
- **Root cause**: Three click/drag handlers cast `e.target` directly to
  `HTMLElement` without checking `instanceof Element`:
  1. `public/js/orders.ts:430` — `setupHanosBtnDelegation()` click handler
  2. `public/js/planner.ts:278` — `batchDragStart()` dragstart handler
  3. `public/js/planner.ts:285` — `batchDragEnd()` dragend handler
  4. `public/js/dishes.ts:521` — `inlineAddAllergenStart()` click handler
- **Severity**: P1. Crash silently breaks the click — user sees nothing happen.
  Two occurrences in 1 day, likely under-reported.
- **Fix**: Add `if (!(e.target instanceof Element)) return;` guard before
  `.closest()` calls.

### B7 — Logout API spam: 2,124 calls in 7 days — **P2**
- **Source**: telemetry aggregation.
- **Pattern**: 1,120 calls on Apr 26, 1,003 on Apr 27, then 1 on Apr 28. All
  with `userId: null, sessionId: null`.
- **Impact**: Wasted API capacity, noise in telemetry. Not a user-facing bug
  but indicates either (a) a bot/crawler hitting `/api/auth/logout`, or (b) a
  frontend redirect loop when sessions expire.
- **Root cause**: Likely the auth check in `public/js/auth.ts` — when a session
  expires, the frontend may call logout and redirect in a loop until the page
  fully reloads. Cross-reference with user complaints about being logged off.
- **Fix approach**: Add rate limiting or deduplicate — if the user is already
  logged out (no session cookie), skip the API call.
- **Severity**: P2. No user-facing symptom besides wasted resources.
